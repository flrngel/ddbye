# UX reset

## Pages

Only two pages exist in the MVP:

1. Landing
2. Console

## Landing purpose

Landing should do only three things:

- explain that the product starts from a messy human brief
- show that diligence happens before writing
- get the user into the console fast

## Console purpose

Console is the whole product.

It has three persistent regions:

1. **Left rail** - saved requests and status
2. **Top composer** - one brief input flow
3. **Main workspace** - research board + outreach studio

## The key UX change

The intake is no longer a long prompt or a campaign/prospect workflow.

It is one clear brief composer with four essentials:

1. Who or what do you want to investigate?
2. What do you want from them?
3. What are you offering?
4. Which deliverable do you want first?

Everything else is secondary.

## Why this is better

### Old shape

Internal model first:

- create campaign
- create prospect
- generate outreach

### New shape

Human intent first:

- describe target
- run diligence
- inspect research
- copy outreach

## Output order matters

The workspace is intentionally ordered as:

1. resolved target
2. why this target
3. context cards
4. recommended wedge
5. evidence
6. subject + final draft

That sequence is the product.

## What was intentionally removed

- campaign page as first-touch UI
- prospect page as first-touch UI
- noisy strategy toggles
- multiple draft modes as the main interaction
- writing-first workflow

## Design system

### Visual tone

- pastel lavender + blue
- soft white surfaces
- rounded components
- high shadow softness
- strong hierarchy without aggressive contrast

### Typography

- rounded heading font
- clean sans body font
- dense but not cramped cards

### Interaction tone

- calm
- trustworthy
- not sales-bro
- not enterprise-grey
