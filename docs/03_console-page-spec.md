# Console page spec

## 1. Left rail

Purpose:

- show prior requests
- let the user jump between them
- prove that the product is organized around requests, not around CRM objects

Each request card should show:

- title
- short parsed hints
- preferred channel
- status
- updated time

## 2. Composer

Purpose:

- collect the entire human brief in one place
- lower the input burden
- avoid asking the user for a machine-perfect prompt

### Fields

#### Target brief

Free text. This is the most important field.

Placeholder should encourage natural input, not prompt engineering.

#### Objective

Free text. The system needs the actual ask to decide what research matters.

#### Offer / context

Free text. The system must know what the sender actually has to offer.

#### Deliverable

Email, LinkedIn DM, or X DM.

#### Goal type

Sell, partnership, fundraise, recruit, advice.

#### Tone

Respectful, direct, warm.

#### Research focus

Optional chips. Useful for steering, not required for success.

## 3. Research board

Purpose:

- make the diligence visible
- justify the outreach
- show the answer before the copy

### Sections

#### Resolved target

Three cards:

- person
- organization
- surface

#### Parsed hints

Small chips that show how the system interpreted the brief.

#### Why this target

Three bullets maximum.

#### Agent run

Compact status steps only. No giant progress theatrics.

#### Context cards

Cards that answer:

- what the agent learned
- what matters on the surface
- what the pitch should sound like

#### Recommended wedge

One clear angle.

Include:

- headline
- rationale
- mention
- avoid

#### Evidence used

Every claim should show whether it came from:

- user brief
- public web
- inference

## 4. Outreach studio

Purpose:

- deliver a sendable output
- keep subject/title visible
- keep channel variations together

### Sections

#### Channel tabs

- Email
- LinkedIn DM
- X DM

#### Subject lines

Only for email.

This must be visible above the body.

#### Final draft

Readable, copyable, and not buried.

#### Follow-up

One low-friction next message.

## UX rule

If the user can copy a good subject and a good draft in under 30 seconds, the page is working.
